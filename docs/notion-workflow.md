## How To Use Notion

Use Notion as the project memory and PM surface.

For meaningful work:

- Search/fetch relevant cards before planning.
- Add concise comments for implementation checkpoints, blockers, release notes, and QA results.
- Update the `Status` property when a card clearly moves state.
- Avoid rewriting old notes unless correcting a factual error. Prefer adding dated comments.

Roadmap status values:

- `Backlog`
- `Needs Elaboration`
- `Ready`
- `In Progress`
- `Blocked`
- `Done`

## What To Record

Record:

- branch/PR links
- Supabase migration names
- production/staging deploy status
- QA evidence and data counts
- blockers requiring Brian action
- decisions that affect future architecture

Do not record secrets, magic links, service-role outputs, or sensitive raw logs.
